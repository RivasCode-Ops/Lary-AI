LARY AI — Guia de Deploy & Piloto

Prepare o MVP para um cliente real em 30 minutos

✓ Pré-requisitos
  • Conta no Render (gratuita) ou Railway
  • Conta no Supabase (gratuita)
  • Chave da OpenAI (gratuita: $5 de crédito inicial)
  • Expo Go instalado no celular
1 Deploy do Backend (Render)
  1. 1 Fazer fork do repositório no GitHub
  2. 2 Conectar ao Render (New Web Service → Connect GitHub repo)
  3. 3 Configurar variáveis de ambiente no Render:
VariávelValor
DATABASE_URLConnection string do Supabase (Passo 2)
OPENAI_API_KEYChave da OpenAI
JWT_SECRETGerar com openssl rand -hex 32
  1. 4 Clicar em Deploy — o Render fará o build e deploy automático
ℹ Dica: O deploy inicial leva de 3 a 5 minutos. Acompanhe os logs para garantir que não há erros.
2 Deploy do Banco (Supabase)
  1. 1 Criar um projeto gratuito em supabase.com
  2. 2 Ir em SQL Editor, copiar o conteúdo de supabase/init.sql e colar → Run
  3. 3 Configurar a DATABASE_URL local e rodar o seed:
# No terminal, na raiz do projeto:
$ export DATABASE_URL="postgresql://..."
$ npm run seed
  1. 4 Anotar a DATABASE_URL e a Anon Key (Settings → API)
ℹ Dica: Salve a Anon Key para usar no frontend (como EXPO_PUBLIC_SUPABASE_ANON_KEY se necessário).
3 Configurar Frontend
  1. 1 No arquivo .env do frontend, atualizar:
EXPO_PUBLIC_API_URL=https://seu-app.onrender.com/api
  1. 2 Rodar o app local:
$ npx expo start
  1. 3 Abrir o Expo Go no celular e ler o QR code
  2. 4 (Opcional) Build para produção com EAS:
$ npx eas build --platform all --profile production

  4. 4 Aprovações → simular aprovação de documento

Testar offline

Ativar Modo Avião no celular e verificar se as funcionalidades offline funcionam (diário, cache de leitura).

5 Coletar Feedback
  1. 1 Após cada uso do cliente, perguntar: “O que travou?”
  2. 2 Anotar o NPS (“De 0 a 10, quanto recomendaria?”)
  3. 3 Registrar os insights no Kanban do projeto (dashboard/index.html)
